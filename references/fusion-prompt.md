# DocParse 퓨전 프롬프트

## 역할
여러 문서 파서의 결과를 비교/조합하여 최고 품질의 단일 마크다운을 생성한다.

## 입력
각 파서의 출력 마크다운이 제공된다:
- **Upstage** (`_upstage.md`): 기준선. 완결성과 노이즈 필터링 최강. 단, 동기 API 100p 제한으로 대용량 문서에서 타임아웃 가능.
- **Gemini** (`_gemini.md`): 텍스트 품질 최상. 단, 30p+ 문서에서 페이지 대량 누락 위험 (최대 68% 누락 사례).
- **LlamaParse** (`_llamaparse.md`): HTML 표 구조 보존력 최강. 대용량 문서에서도 페이지 완전성 우수.
- **Mistral** (`_mistral.md`): 페이지 완전성 우수, 대용량 문서의 교차 검증 보장. 한국어 워터마크 환각 주의.

## 퓨전 원칙

### 1. 구조 기준선: Upstage
- Upstage 결과를 문서의 뼈대(heading 구조, 페이지 순서)로 사용
- 다른 파서에만 있는 섹션이 발견되면 Upstage 결과의 적절한 위치에 삽입

### 2. 텍스트 품질 보강: Gemini
- Gemini가 커버하는 구간에서, 문장 표현이 Upstage보다 자연스럽고 정확하면 Gemini 버전 채택
- Gemini 누락으로 마킹된 구간은 무시 (Upstage 유지)
- 판단 기준: 문법적 자연스러움, 고유명사 정확도, 문맥 완결성

### 3. 표 구조 보강: LlamaParse
- LlamaParse의 HTML 표(`<table>`)가 Upstage의 마크다운 표보다 정확하면 채택
- HTML 표는 마크다운 표로 변환하여 출력
- `colspan`/`rowspan`이 있는 복잡한 병합 셀은 평면 마크다운으로 단순화 (구조 정보 일부 손실 허용)
- 빈 셀, 셀 정렬, 헤더 행은 최대한 보존

### 4. 교차 검증
파서 간 불일치 발견 시:
- **고유명사/수치/데이터**: 다수결 채택. 동률이면 Upstage 우선.
- **누락 콘텐츠**: 2개 이상 파서에 있으면 포함.
- **환각 의심**: 1개 파서에만 있고 다른 파서 모두에 없는 콘텐츠는 제외.

### 5. Mistral 역할 (전 언어)
- 교차 검증 보조 소스로 활용. Upstage/Gemini 실패 시 LlamaParse와 함께 **주 소스로 승격**
- Mistral 단독으로 콘텐츠 채택하지 않음 (환각 위험)
- LlamaParse + Mistral 일치 시: 두 파서가 독립적으로 동일 콘텐츠를 추출한 것이므로 신뢰도 높음 → 채택
- 한국어 문서의 Mistral 워터마크 환각 (`höncom`, `한국예법원처` 등)은 노이즈로 제거

## 출력 형식

- 순수 마크다운 (HTML 태그 불가)
- heading 계층: 원본 문서의 구조를 충실히 반영
- 표: 마크다운 표 문법 사용
- 이미지: 파서 출력의 이미지 참조/캡션은 보존하되, 깨진 링크는 제거
- 페이지 구분: 불필요 (연속된 하나의 문서로 출력)

## 품질 체크리스트
퓨전 완료 후 자체 검증:
- [ ] 기준선 파서의 모든 heading이 최종 결과에 존재하는가?
- [ ] 표의 행/열 수가 원본과 일치하는가? (LlamaParse 또는 Mistral 참조)
- [ ] 고유명사/수치가 파서 간 교차 검증되었는가?
- [ ] Gemini 누락 구간이 다른 파서로 올바르게 대체되었는가?
- [ ] 환각으로 의심되는 콘텐츠가 제거되었는가? (Mistral 워터마크 환각 포함)
- [ ] 온전한 파서 결과가 최소 2개 이상으로 교차 검증이 이루어졌는가?
