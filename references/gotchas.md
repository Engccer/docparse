# 주의사항·에러 처리·동기화

## 주의사항 (실제 사례 기반)

- **포맷 품질 ≠ 데이터 정확도**: 깔끔한 마크다운 표가 정확한 숫자를 보장하지 않는다. Gemini·LlamaParse v2 모두 스캔 수기 양식에서 열 배정 오류 발생.
- **Gemini·LlamaParse v2는 스캔 수기 양식 표의 Primary·교차검증에 부적합**: Upstage·Mistral 2파서가 일치하는 값을 채택.
- **합본 PDF보다 개별 PDF 파싱이 정확**: 동일 양식 다수를 합본하면 열 이동 오류가 증폭.
- **산술 검증만으로는 열 이동을 탐지할 수 없음**: Step 5.3 셀 위치 비교가 유일한 방어선.
- **보조 파서의 표가 지저분해도 숫자 추출은 시도할 것**: 포맷이 나빠도 숫자 값은 정확할 수 있음.
- **스캔 이미지 + 수기 표 = 최고 난이도**: 최소 3개 파서 필수. 단일 파서 신뢰 불가.
- **시험지·고사 원안·평가지 = small 티어라도 다파서 교차검증 필수**: 옵션 텍스트 한 글자가 정답을 좌우. Gemini 단독 시 옵션 순서·원숫자(ⓐ→ⓠ) 오인식 발생. 권장: Gemini + Upstage + ODL + 정답표 매칭.
- **PDF Read 시각 렌더링 = 소형 PDF의 ground truth**: ≤20p PDF는 Claude Code의 `Read` 도구로 시각 이미지 직접 읽기 가능 → 파서 cross-validation의 최종 결정자로 사용.
- **LlamaParse v2 크레딧 관리**: `python llamaparse_parse.py --credits`로 잔여 확인. 무료 월 10,000크레딧, 롤오버 없음. 갱신일 전 소진 권장.
- **ODL은 텍스트 레이어 있어도 본문 숫자 누락 가능**: 출판 임베딩 폰트의 글리프 매핑이 깨지면 ODL이 본문 숫자/특수문자를 전부 빈칸으로 출력. `has_text_layer: true`만 보고 ODL Primary 채택하지 말 것. 본문 첫 문단 1~2개 숫자를 v2/Upstage와 즉시 대조 필수 (2026-05 학술 보고서 554p에서 실측).
- **학술/연구 보고서에서 v2 헤딩 위계 불안정**: 같은 깊이 절(`1.`, `2.`, `3.`)에 디자인 크기 따라 무작위로 H1/H2/H3 부여. 인쇄용 책자와 달리 학술 보고서에서 빈번. 본문 절 위계 전체 정규화는 가성비 낮음, 챕터 H1 통일과 표 캡션 일관성만 정리.
- **수기 자유 서술에서 v2 환각은 "그럴듯한 단어 단위"로 발생**: 인명 생성("Dear me"→"Dear Mr. Han"), 접속어 삽입("the end"→"In the end"), 오기 정규화(shoud→should). 산술·구조 검증으로 못 잡고 시각 대조만이 방어선. 학생 오기가 채점 요소인 답안지에서는 v2의 유창한 출력일수록 의심 (2026-06-11 실측, tier-rules "수기 학생 답안" 참조).
- **다페이지 시각 판독은 서브에이전트에 위임**: 메인 컨텍스트에서 페이지 PNG 수십 장을 직접 Read하면 이미지 누적 한도에 걸려 **이후 새 이미지 첨부가 전부 거부**될 수 있음(특히 폭 2000px 초과 이미지가 섞이면 가속). 절차: ① PyMuPDF로 영역 크롭 생성(폭 1500px 이하 권장) ② 페이지 범위를 3~4분할해 서브에이전트(general-purpose)에 판독 위임 ③ 잔여 쟁점 글자는 같은 에이전트에 SendMessage 후속 질의로 8~14배 확대 재판독. 에이전트가 직접 PyMuPDF로 자기 크롭을 만들게 하면 더 효율적 (2026-06-11 실측).

## 에러 처리

- 파서 실패/타임아웃 → 나머지로 진행 (최소 1개 성공 필요).
- 1개만 성공 → 해당 결과를 노이즈 정리 후 `_fused_v3.md`로 저장, 교차 검증 불가 경고.
- LlamaParse v2 타임아웃 (xlarge) → ODL 폴백 또는 Upstage로 대체.
- Upstage 비동기 타임아웃 → 교차검증만 불가, Primary(v2)에 영향 없음.
- 모든 파서 실패 시 사용자에게 오류 보고.

## Claude Code 스킬로 설치

이 패키지를 `~/.claude/skills/docparse`(심링크 또는 junction)로 연결하면 `/docparse` 슬래시 명령으로 동작한다. 자기 완결형 패키지이므로 `SKILL.md`, `parsers/`, `scripts/`, `references/` 폴더 하나만 연결하거나 복사하면 모든 환경에서 완전히 동작한다.
