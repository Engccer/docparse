# 손글씨 충실 전사: Vision→Claude 비전 캐스케이드 (최고 충실)

> 5개 subset 실측으로 확정(2026-06-17, 1학년 영어 쓰기 수행평가 4개 학급 합본 약 80명). 손글씨 답안을 **오기 보존(diplomatic transcription)** 으로 전사할 때 가장 충실한 방법과 모델 선택.
>
> **정량 재확인(2026-06-23)**: 7개 학급 전부의 **수동 검증 정본**을 ground truth로 삼아 3단 토너먼트(학급 A: Sonnet단독 vs Opus단독 / 학급 B: GV+Sonnet vs GV+Opus / 학급 C: 단독 vs 캐스케이드)를 CER·WER로 정량 측정. 결론을 수치로 확정: **GV+Opus 캐스케이드가 챔피언**(정본 대비 문자 충실도 ~98.2%, 차이 ~1.8%), 단독 비전(~97%)을 R3에서 +1.45%p로 앞섬.

## 모델 비교 결론 (충실도 순위)

**Vision 캐스케이드(Opus repair) > Claude 단독 비전(Opus ≥ Sonnet) > [탈락] Google Vision 단독 / Gemma·Haiku 로컬 VLM**

- **Claude Opus 4.8 ≥ Sonnet 4.6 (단독 비전)**: 이미지 직접 판독. 충실·읽기순서 정상·환각 거의 0. 단 *가장 어려운 오기*를 가끔 윤문(speacial→special, eldery→elderly, frenads→friends, guiter→guitar, didn4→didn't)하고 **매우 지저분한 페이지·장문을 garble/환각**. **정량(2026-06-23, 정본 대조): 단독 비전에서 Opus가 Sonnet보다 문자 +1.01%p(학급 A, 21명 중 18명 우위)로 박빙이 아니라 일관 우위.** 캐스케이드에서는 격차가 +0.33%p로 좁아진다(Vision 드래프트가 철자를 대신 확정하므로 모델 기여분 감소). **속도 우위·1콜·Vision API 불필요.** 단, messy·장문에서 confabulation 위험(아래 §정량 검증 장문 사례).
- **Google Vision 단독**: 비-LLM 리터럴 OCR. 오기 보존이 가장 literal(cookles·dras·Roblox·specill·eldery 등 LLM이 윤문한 것까지 보존)·환각 0·단어별 confidence 제공. **그러나 읽기순서 뒤섞임·단어 붙음·일부 글자오독으로 단독 사용 불가.** 결정론적 geometry 재정렬(모델 0)로도 구조 문제 해결 못 함(폼 이중검출 중복·잔여 뒤섞임).
- **Gemma 4 12B / Haiku 4.5 (로컬·소형 범용 VLM)**: **탈락.** 윤문 + **환각(런마다 불안정: Roblox→Wumpus→yummy chicken, 인사말→favorite saying/auntie)** + 누락. Haiku는 빈칸을 그럴듯하게 메움(play the drums→tenboulin, stay health→stay healthy and God bless). 동일·강한 프롬프트로도 안 잡힘(모델 고유 성질).

→ **Vision(리터럴 철자 + confidence)과 Claude 비전(구조·가독성)은 상보적.** 둘을 캐스케이드로 결합한 것이 최고 충실.

## 정량 검증 (2026-06-23 토너먼트, 정본 대조 CER/WER)

3개 학급 정본을 ground truth로 3단 비교(학번·이름은 동일 주입, 영어 본문 전사만 측정, micro=길이가중).

| 라운드 | 대결 | 승자 | 문자 정확도(micro) | 학번별 승 |
|--------|------|------|------:|------|
| R1 (학급 A) | Sonnet단독 vs Opus단독 | **Opus단독** | 96.84% → 97.85% | 18:2 |
| R2 (학급 B) | GV+Sonnet vs GV+Opus | **GV+Opus** | 97.82% → 98.15% | 11:6 |
| R3 (학급 C) | Opus단독 vs **GV+Opus 캐스케이드** | **🏆 캐스케이드** | 96.74% → 98.19% | 13:6 |
| 복제 (학급 D) | Opus단독 vs GV+Opus 캐스케이드 | 캐스케이드 | 98.03% → 98.27% | 14:6 |

- **챔피언 = GV+Opus 캐스케이드**: 정본 대비 문자 차이 ~1.8%(각 1.85%·1.81%·1.73%), 단어 정확도 ~91~92%. 세 학급에서 ~98%로 안정.
- **복제 검증(학급 D, 4번째 독립 학급)**: 방법 결론 재현(캐스케이드 문자·macro 우위, 학번별 14:6). 단 격차는 +0.24%p로 학급 C(+1.45%p)보다 작고 단어-micro는 단독 근소 우위 → **격차 크기는 messy·장문 비율에 비례**(학급 D엔 극단 장문 부재). 함의: **캐스케이드는 no-regret 선택**(깔끔하면 단독과 동률, messy·장문이면 큰 이득, 문자 정확도가 단독보다 나빠지지 않음). 채점처럼 최악 케이스 충실도가 중요하면 캐스케이드 기본값.
- **캐스케이드 우위 메커니즘(최장문 ~1385자 답안)**: 단독 87.5% vs 캐스케이드 96.5%. 단독은 흘림·밀집 구간을 **그럴듯하게 환각**(원문에 없는 매끄러운 어구로 메움)했고, 캐스케이드는 Vision literal 드래프트가 **실제 단어에 닻**을 내려 억제했다. 단독 비전이 messy·장문에서 무너지는 지점을 드래프트가 구조적으로 막는다(verbatim 사례는 비공개 보고서).
- **모델 선택**: 충실 최우선 → Opus. 비용 절감 → 캐스케이드 repair에 한해 Sonnet 허용(품질 손실 ~0.3%p).

## 두 가지 모드 (용도별)

| 용도 | 방법 | 비고 |
|------|------|------|
| **채점·저숙련 답안·충실 critical** | **Vision 캐스케이드** (아래) | 5/5 subset 최고. messy 답안 복구력이 결정적 |
| **속도 critical 대량·요지면 충분** | **Claude 단독 비전** (이미지만, diplomatic 프롬프트) | 1콜·빠름. 단 가장 어려운 오기 윤문 가능 |

## Vision 캐스케이드 워크플로우

1. **Vision OCR**: `python parsers/gvision_parse.py <page>.png --raw < /dev/null` (페이지별). 리터럴 텍스트 + 단어별 confidence + geometry JSON(`--raw`).
2. **영어 본문 드래프트 추출**: Vision `_gvision.md`에서 양식 헤더(`## p.`)와 체크리스트(`자기 점검표`/`저신뢰:`) 사이의, **라틴 글자 포함·한글 미포함 라인**만 모아 페이지별 드래프트로(한글 양식 자동 제거).
3. **Claude 비전 repair**: 각 페이지의 **이미지 + Vision 드래프트**를 Claude(Sonnet 권장·저비용, Opus 동급)에 주고 아래 프롬프트로 교정. 합본은 페이지 대량 Read 회피 위해 서브에이전트에 위임(`gotchas.md`). 합본 서브에이전트 결과는 알림이 잘릴 수 있으니 **Write 도구로 파일 저장**시켜 회수.
4. **충실 조립**: 결과를 fused로. 모호 글자 채택 근거는 "판독 노트" 부록.

### Repair 프롬프트 (핵심 원리: 이대로 주입)

```
손글씨 학생 영어 답안(채점용)을 충실 전사하는 캐스케이드 repair 단계다.
입력 = (a) 원본 이미지, (b) Google Vision OCR 드래프트(리터럴: 학생 철자/오기를 정확히
보존하나 읽기순서 뒤섞임·단어 중복·일부 글자오독·한글 양식 포함).

규칙:
- 이미지 = 읽기순서·줄바꿈·단어경계·Vision 중복 제거의 권위.
- ★Vision 드래프트 = 철자의 기본 권위. 모든 단어의 철자를 드래프트 그대로 유지하라.
  학생 오기를 표준 영어로 정규화하는 것은 오류다
  (speacial→special, eldery→elderly, frenads→friends, hapy→happy, studid→studied 금지).
- 드래프트 철자를 바꾸는 것은 이미지가 "명백히 다른 글자"를 보일 때만
  (Vision이 proud→proad, rice→rise, 이름 등 글자를 오독한 경우 이미지의 실제 글자로 교정).
- ⚠ 디짓 섞인 오기(didn4, couldn7 등): Vision은 4를 t로 오독해 "didn't"로 정규화하기 쉽다.
  이미지에서 숫자 자형이 보이면 드래프트보다 이미지를 우선해 didn4 그대로 둔다
  (이 케이스는 Vision 드래프트가 틀리므로 철자-권위 규칙의 예외).
- 무단 교정·환각·요약 금지. 영어 편지 본문만(인사말~서명). 한글 양식 무시. 한글 서명은 보이는 그대로.
- ★출력에는 학생이 쓴 글자만. 어떤 주석·메모·괄호 설명도 본문에 넣지 말 것((주의…), [취소선], 학번 코멘트 금지). 학번 불일치 등 메타는 본문이 아니라 별도 보고로.
- 취소선으로 명확히 그은 부분은 표시 없이 조용히 생략.
- 정말 판독 불가능한 한 글자는 물음표 하나 `?` 로만(설명·[illegible] 금지).
- 백지·골격만(Dear/Best,만) 있으면 본문을 비운다(채점 정본 관례 = 백지).
```

> **클린 출력 규칙은 채점 정량화의 전제다(2026-06-23).** 1차 실행에서 에이전트가 본문에 `[취소선]`·`(주의:…)` 메모를 달아 짧은 답안의 CER을 오염시킴 → 위 규칙으로 통일. 정본도 `[미작성]`·`(빈 줄만…)` 같은 편집 주석을 쓰므로, 채점 시 `[…]`와 한글 포함 `(…)`를 양쪽에서 대칭 제거하고 **백지(0자)는 집계 제외**한다(0자 정본은 CER 0/100%만 나와 macro 왜곡).

## 잔여 약점 / 주의

- **didn4류 디짓-인-워드**: Vision이 못 읽으면 캐스케이드도 놓치기 쉬움 → 위 프롬프트의 예외 규칙 + 시각 판독으로 보강(Claude 단독 비전이 오히려 이 토큰을 잡기도 함).
- **PII**: 학생·환자 등 민감 손글씨는 Vision(클라우드) 전송 정책 확인. 완전 로컬 필요 시 `handwriting-ocr-engines.md`의 로컬 엔진(단 confidence/충실도 트레이드오프).
- **정본 역검증 부산물**: 독립 다(多)판독이 일치하는데 기존 정본과 다르면 정본 누락 의심(실측: `too difficult`/`too tired`를 3모델이 일치 판독 vs 정본 누락).
