# 후처리 패턴 (Step 4 / Step 7)

`_llamaparse.md` 또는 `_opendataloader.md`를 `_fused_v3_<파서조합>.md`(Primary 파서명으로 시작, 예 `_fused_v3_llamaparse.md`)로 복사한 뒤 LLM 교차 검증 전에 적용하는 정규식·구조 정리 패턴 모음. 파일명 규칙은 SKILL.md Step 4 참조.

## LlamaParse v2 Primary 후처리 (medium~xlarge 기본)

v2 출력은 구조적으로 깔끔하지만(노이즈 적음, 헤딩 양호, 목차 정리됨) **인쇄용 책자·다이어그램 풍부 문서**에서는 아래 잔여 노이즈가 출현. 텍스트 위주 문서(법률·논문·보고서)에서는 1번만 처리하면 된다.

### 1. 이미지 placeholder 처리

- **시각장애인 접근성 critical 문서**(장애인 안내자료, 매뉴얼, 교육 자료): `scripts/generate_alt_text.py`로 한국어 상세 alt text 자동 생성 후 inline 보존. PyMuPDF로 PDF 페이지 렌더링 + Gemini Vision API. 매핑 JSON 저장하여 재실행 시 재사용 가능.

  ```bash
  python <스킬루트>/scripts/generate_alt_text.py \
      --pdf doc.pdf --markdown doc_llamaparse.md \
      --output doc_with_alt.md --map alt_map.json
  ```

  **`--skip-korean` 옵션 사용 금지(기본 권장)**: LlamaParse v2가 한국어 텍스트 OCR로 alt를 만들 때 너무 짧게(`법령 아이콘`, `접근로 사진`, `복도 사진`) 출력하는 경향. 시각장애인이 정보를 얻기에 무의미 → 한국어 alt가 있어도 Gemini로 재생성하여 시각적 특징(색·구성·어떤 법령인지·어떤 편의시설 특징인지)을 풍부하게 풀어내야 함. 2026-05 인사관리안내서 133p에서 LlamaParse 한국어 alt 113개 중 다수가 4~10자 이내였음.

- **그 외 일반 문서**: `![alt](page_N_image_N_v2.jpg)` → alt가 본문 정보 담은 경우만 `(이미지: alt)` 보존, 단순 장식은 제거. `NO_CONTENT_HERE`도 일괄 제거.

### 2. `` ```description ... ``` `` 코드 블록 제거

v2가 페이지 색상·디자인 요소를 description 블록으로 출력하는 경우. 본문 정보 아님 → 제거.

### 3. 반복 페이지 머리글 제거

인쇄용 책자의 페이지 헤더(`<문서 제목> 안내자료` 등)가 페이지마다 반복 추출됨. 첫 표지 H1만 남기고 본문 중복 라인 제거.

### 4. 챕터 표지+본문 첫 페이지 H1 중복 통합

v2는 챕터 분리 표지(`# Ⅱ` 단독)와 본문 첫 페이지 헤딩(`# Ⅱ 장애인교원...`)을 모두 추출하여 H1이 2~3중으로 쌓임. 단일 H1으로 통합 + `P·A·R·T N` 표지 라벨 제거.

**연속 챕터 H1 미니 TOC 패턴**도 함께 처리: 챕터 H1이 빈 줄 사이에 3개 이상 연속으로 나오는 영역(부록 페이지의 챕터 목록을 본문에 잘못 임베드한 경우)은 미니 TOC로 판정하여 일괄 제거. 정규식 예시:

```regex
(?:^# (?:장애인교원의 이해|장애인교원 인사관리|장애인교원 편의지원|부록)\s*\n+){3,}
```

2026-05 인사관리안내서 133p에서 본문 진입 H1 3건(line 136/683/1566)은 유지하고 미니 TOC 3건만 제거하는 게 정확.

**분할·합본 챕터 H1 패턴** (2026-05-14 해당 안내자료 148p에서 4건 관측): 표지 디자인이 큰 로마숫자(또는 영문 IV)를 좌상단에, 한글 챕터명을 우측에 배치하면 v2가 한 챕터의 H1을 2~4행으로 쪼개 추출함.

```
# Ⅲ
# 복무, 성과 및 인사관리 지원
# Ⅲ 복무, 성과 및 인사관리 지원
```

또는:

```
# IV
장애인교원
권리옹호 지원
# Ⅳ 장애인교원 권리옹호 지원
```

합본 H1만 남기고 나머지(단독 로마숫자, 분리된 한글 줄, 영문 IV)는 모두 제거. 후속 `P·A·R·T N` 라벨도 제거(부록 PART 번호와 본문 챕터 번호가 어긋날 수 있음. 해당 안내자료에서는 Ⅳ 챕터 표지에 `P·A·R·T 5`가 붙어 있었음).

**표지 페이지의 책 제목 시각 디자인 4행 분리** (2026-05-14 학술 보고서 554p에서 5건 관측): 챕터 표지 페이지의 책 제목이 시각적으로 4줄로 배치되면 v2가 그대로 4행 추출:

```
장애유형별
장애인교원
근무 지원
방안

# 장애인교원 지원 관련
# 선행연구 분석
P·A·R·T **Ⅱ**

# 장애인교원 지원 관련 선행연구 분석
```

5개 챕터 모두 동일 패턴이면 일괄 Edit 가능. 책 제목 4행 + 분리된 H1 2~3행 + `P·A·R·T <라벨>` + 합본 본문 H1 전체 블록을 단일 `# Ⅱ. 장애인교원 지원 관련 선행연구 분석`로 통합.

**`P·A·R·T # <챕터제목>` 인라인 합본** (2026-05-14 동일 보고서에서 2건 관측): 일부 챕터 표지에서 P·A·R·T 라벨과 챕터 본문 시작 H1이 한 줄에 합쳐져 나옴.

```
# Ⅳ. 장애유형별 장애인교원 근무지원 방안 탐색
P·A·R·T # 장애유형별 장애인교원 근무 지원 방안 탐색

## 1. 지체장애 및 뇌병변장애인
```

`P·A·R·T # ...` 줄 통째로 제거.

**표 경계 페이지 헤더가 H1으로 오탐** (2026-05-14 보고서에서 1건 관측): 동일 표가 페이지 경계로 끊겼다 이어지는 부분에 페이지 헤더 라벨(`Ⅲ. 교원의 직무분류표 개발 및 타당성 검증`)이 H1으로 추출됨.

```
| C. 교무학사 | 1. 학교교육과정 운영계획 수립하기 | 학교 교육과정 편성, 운영위원회 조직 및 운영 |

# Ⅲ. 교원의 직무분류표 개발 및 타당성 검증

| 책무(duty) | 과업(task) | 과업요소(task elements) |
```

표 행 사이에 본문 챕터 H1이 등장하면 페이지 헤더 노이즈로 의심 → 제거하여 표 연결성 복원. 표 안의 페이지 헤더는 본문 첫 등장이 아니라 챕터 중간에 동일 H1이 다시 나타나는 패턴으로 식별.

**헤더 inline에 박힌 디자인 아이콘 alt 제거**: 챕터·섹션 시작의 색상 사각형/원형 안에 박힌 숫자 아이콘이 LlamaParse v2 출력에서 헤더 텍스트 앞에 inline으로 박힘.

```
## (이미지: 자주색 사각형 안에 흰색 숫자 '1'...) 1 장애인교원의 정의 및 특성
# (이미지: 보라색 장식용 아이콘...) 참고문헌
```

헤더 가독성을 해치고 정보 가치 없음 → 정규식 `^(#+) \(이미지:[^)]*\) (.+)$` → `\1 \2`로 치환. `accessible.txt`를 `--instructions`로 주입하면 이 패턴이 줄어들지만 완전히 사라지지는 않으므로 후처리는 여전히 필요.

### 5. mermaid 직후 동일 이미지 placeholder 중복 제거

v2가 같은 흐름도를 mermaid 코드 블록과 이미지 placeholder로 동시 출력하는 케이스(학습활동지 배부 흐름도에서 3중 중복 관측). mermaid를 보존하고 인접 placeholder만 제거.

### 6. 챕터 로마숫자 OCR 오류 점검

≤5개 챕터 문서에서 1~2건 빈도. 목차의 챕터 번호가 본문 P·A·R·T N 표기와 어긋날 수 있음. 본문 P·A·R·T 라벨을 ground truth로 삼아 검증 (예: 청각장애인교원 챕터가 PART 5인데 v2/Upstage 모두 Ⅳ로 추출 → 본문에서 `P·A·R·T 5` 확인 후 Ⅴ로 정정).

`pdftoppm` 미설치 환경에서 PDF Read 도구 사용 불가 시, **PyMuPDF의 `page.get_text()`로 PART/Ⅴ 등 영문·로마숫자 직접 추출**하여 챕터 번호 검증 가능 (한글 인코딩 깨져도 영문 P·A·R·T·숫자는 정확).

### 7. 다이어그램에만 정보 있는 페이지 보강

인포그래픽/플로우차트로만 표현된 본문(예: "수업 시작 전/직후/중 기자재 목록" 단계별 구분)은 v2 placeholder가 정보를 누락. Upstage figcaption description에서 텍스트를 추출해 본문 표·리스트로 변환 (OCR 오류 보정 필요: `각중→각종`, `익기→악기` 등).

### 부가 패턴: 챕터 표지 `°` 단독 노이즈

챕터 분리 표지 페이지의 디자인 점이 v2 출력에서 `°` 한 줄로 잡힘(해당 책자 148p에서 2건). 단독 줄로 등장하면 후처리에서 제거.

### 부가 패턴: `<page_number>` 페이지 경계 태그 제거

v2가 페이지 경계 번호를 `<page_number>N</page_number>` 태그(번호가 별도 줄에 오는 멀티라인 형태 포함)로 출력하는 경우. 본문 정보 아님 → 제거. 2026-06-25 교육과정 비교 리포트 자료 10개 문서에서 총 76개 관측(별책16 12·CSP 프랑스어 원문 25·S10 10 등 텍스트 위주 행정/학술 문서에 빈발). 정규식:

```regex
\s*<page_number>.*?</page_number>\s*
```

`re.DOTALL`로 치환 후 `\n{3,}` → `\n\n` 빈 줄 재압축. 교육과정 인용 시 쪽수는 원본 PDF로 참조하므로 본문 태그는 제거가 안전.

## ODL Primary 후처리 (폴백, 크레딧 부족 시)

`normalize_odl.py`로 일괄 처리:

```bash
python <스킬루트>/scripts/normalize_odl.py "<odl_output.md>" "<파일명>_fused_v3_opendataloader.md"
```

스크립트가 자동 처리하는 항목:

1. **페이지 구분자 제거**: `---`, `<!-- Page N -->` 행 삭제.
2. **이미지 placeholder 제거**: `![image](...)`, `<figure>/<figcaption>` 블록 삭제.
3. **h6 heading 정규화**: 번호 패턴 기반 계층화 (Ⅰ~Ⅵ→H1, N.→H2, N)→H3, (N)→H4, ①→H5).
4. **plain-text heading 승격**: blank line 사이의 standalone 번호 패턴을 heading으로.
5. **챕터 표지 페이지 제거**: `# [로마숫자]` + 제목 + mini-TOC 패턴 감지 시 삭제.
6. **반복 문서 제목 제거**: 표지 잔재.
7. **과다 빈 줄 압축**: 연속 3줄 이상 → 2줄.

## Step 7: 최종 노이즈 정리 (Primary 무관)

LLM이 최종 fused 파일(`_fused_v3_<파서조합>.md`)을 최종 점검하며 남은 아티팩트 제거:

- **OCR 아티팩트**: `一`, `□` 등 특수문자 오인식.
- **페이지 경계 고아 줄**: 문장이 잘린 채 남은 짧은 줄.
- **중복 heading**: 페이지 경계에서 반복.
- **환각 텍스트**: Mistral의 한국어 오인식, 중국어 텍스트 등.
- **이미지 placeholder 및 figcaption 블록**: `![image](...)`, `<figcaption>...</figcaption>`.
- **Upstage OCR 단어 분절** (스캔 문서, Upstage 보조 사용 시): `Negat ive`→`Negative`, `Ur ine`→`Urine` 등. LlamaParse v2 Primary 사용 시 이 문제 없음(v2는 OCR 정확도 A).
- **의학 약어 오인식** (Upstage): `C0L2A1`→`COL2A1`, `Rube IgG`→`Rubella IgG` 등. v2 Primary 시 정확.
- **HTML 체크박스 아티팩트**: `<label><input type="checkbox">` → `□`.
- **반복 페이지 푸터** (출력자/전자서명자/원본대조필 블록): 첫 번째만 유지하거나 압축.
